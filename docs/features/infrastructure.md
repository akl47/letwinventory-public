# Feature: System Infrastructure

## Context
Letwinventory is a web-based enterprise resource application for manufacturing workflow management. The infrastructure defines the foundational architecture decisions including client-server separation, containerized deployment, automated CI/CD, database backup strategy, environment configuration, and data integrity patterns like soft deletion. These decisions enable all other features to operate reliably across development and production environments.

### Client-Server Architecture (Req #94)
- **Frontend:** Angular 19 with standalone components, signals, and Angular Material
- **Backend:** Node.js/Express 5 with Sequelize ORM
- **Database:** PostgreSQL (production), SQLite (tests)
- **Communication:** REST API at `/api/*`

### Environment Configuration (Req #95)
- `.env.development` / `.env.production` loaded based on `NODE_ENV`
- Backend: `backend/config/config.js` reads from env files
- Frontend: `environment.ts` (dev: `http://localhost:3000/api`), `environment.prod.ts` (prod: `https://letwinventory.letwin.co/api`)
- Angular `fileReplacements` in `angular.json` swaps env files per build config
- E2E uses port 4201 with proxy to backend on 3000

### Soft Deletion (Req #2)
- All models with `activeFlag` field (BOOLEAN, default: true)
- DELETE endpoints set `activeFlag = false` rather than removing records
- Queries filter by `activeFlag: true` unless explicitly including inactive
- "Show Inactive" toggles in UI list views

### Data Integrity — ALCOA (Req #131)
- All records include `createdAt` and `updatedAt` timestamps (auto-generated by Sequelize)
- All user-facing actions record `userID` from authenticated session
- No mechanism to modify timestamps retroactively
- Soft deletion prevents data loss

## API Contracts

### No dedicated infrastructure endpoints
Infrastructure is cross-cutting. The following patterns apply to all API endpoints:

- **Authentication:** `checkToken` middleware on all non-auth routes
- **Permissions:** `checkPermission(resource, action)` middleware after authentication
- **Soft delete:** DELETE endpoints set `activeFlag = false`
- **Timestamps:** Sequelize auto-manages `createdAt`/`updatedAt`
- **Error responses:** 401 (unauthenticated), 403 (forbidden), 404 (not found), 400 (validation)

## UI Design

### No dedicated infrastructure UI
Infrastructure manifests through:
- **Login page** (`/home`) — guest guard redirects authenticated users to `/tasks`
- **Sidebar navigation** — collapsible, auto-collapse on mobile (`≤768px`)
- **Error notifications** — Material snackbar via `ErrorNotificationService`
- **"Show Inactive" toggles** — available on all list views for soft-deleted items

## Implementation Notes

### Key Files
- `backend/config/config.js` — database configuration from env vars
- `backend/models/index.js` — auto-discovers models from subdirectories
- `backend/api/index.js` — auto-discovers API routes from subdirectories
- `backend/middleware/checkToken.js` — JWT authentication middleware
- `backend/middleware/checkPermission.js` — permission enforcement middleware
- `backend/middleware/bodyValidator.js` — request body validation
- `docker-compose.prod.yml` — production Docker config
- `.github/workflows/` — CI/CD pipelines
- `scripts/backup-db.sh` — daily pg_dump with change detection, Gmail SMTP notifications
- `scripts/pull-backup.sh` — off-site backup pull from VPS via SCP
- `scripts/deploy-update.sh` — health check with retries

### Patterns Followed
- Auto-discovery for models and routes (directory scanning)
- Environment-specific configuration via dotenv
- Soft deletion across all user-created data
- ALCOA data integrity (Attributable, Legible, Contemporaneous, Original, Accurate)

### Edge Cases
- SQLite used in tests lacks PostgreSQL sequences, `Op.iLike`, and raw SQL unions — these tests are skipped
- Docker container auto-rebuilds on file changes in development
- Backup script skips if no database changes detected (via `pg_stat_user_tables` hash)
- Backup script skips if disk usage >= 90% full
